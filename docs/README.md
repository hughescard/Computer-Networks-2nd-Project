# Directorio `docs/`

Aquí se encuentra la **documentación** del proyecto.

## Contenido esperado

Algunos documentos que se crearán a lo largo del desarrollo:

- `arquitectura.md`: descripción de los módulos lógicos del sistema y cómo interactúan.
- `topologia.md`: topología de red del laboratorio (portal, clientes, IPs, máscaras, rutas).
- `firewall.md`: reglas de firewall, scripts usados y explicación.
- `desarrollo.md`: cómo preparar el entorno de desarrollo y ejecutar el sistema.
- `logs.md`: formato y ubicación de los archivos de log.
- `antisuplantacion.md`: enfoque para el control de suplantación de IP/MAC.
- `checklist_requisitos.md`: relación entre requisitos del enunciado y su implementación.
- Otros documentos que vayan siendo necesarios (pruebas, diseño de UX, etc.).

## Convenciones

- Todos los documentos en **Markdown** (`.md`).
- Explicar aquí las decisiones importantes de diseño para que alguien nuevo pueda entender el proyecto.
- Desde el `README.md` raíz se deben referenciar los documentos clave de este directorio.
